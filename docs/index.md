# did-btcr2-js — Contributor Documentation

This is the contributor-facing documentation for `did-btcr2-js`, the TypeScript reference implementation of the [`did:btcr2`](https://dcdpr.github.io/did-btcr2/) DID method.

**Looking for user-facing documentation?** This site is for people working *on* the codebase. If you want to install, configure, or use `did:btcr2` in your own application, go to [btcr2.dev](https://btcr2.dev/).

## Audience

This documentation is for:

- **New contributors** trying to orient themselves in the monorepo
- **Maintainers** working on architecture, build, or release tooling
- **Reviewers** verifying changes against the project's design intent
- **Spec implementers** in other languages (Java, Python, Rust) who want to compare against the TypeScript reference

## Repository layout (high level)

```
did-btcr2-js/
├── packages/
│   ├── common/        — types, canonicalization, hashing, JSON patch, errors
│   ├── keypair/       — secp256k1 key pairs, BIP340 Schnorr signatures
│   ├── cryptosuite/   — bip340-jcs-2025 Data Integrity proof suite
│   ├── bitcoin/       — Bitcoin Core RPC + Esplora REST clients (sans-I/O)
│   ├── kms/           — key management interface and default implementation
│   ├── smt/           — optimized Sparse Merkle Tree
│   ├── method/        — core did:btcr2 implementation (resolver, beacons, updater)
│   ├── api/           — high-level SDK facade over method
│   └── cli/           — commander.js CLI binary wrapping api
├── docs/              — this documentation (committed source)
├── tsconfig.base.*    — shared TypeScript configuration
├── eslint.config.cjs  — shared linter configuration
└── package.json       — monorepo root with workspace + build scripts
```

## How this site is built

This site is generated by [TypeDoc](https://typedoc.org) from two sources:

1. **Hand-written narrative documentation** — the markdown files under `docs/` (architecture, contributing guides, ADRs). Pulled in via TypeDoc's `projectDocuments` option.
2. **Auto-generated API reference** — extracted from TypeScript source comments in each package's `src/`.

Both are merged into a single navigable site. Search works across both narrative and API content.

The build output lives in `.docs-site/` at the repo root and is **not committed** — it's a build artifact. Run `pnpm docs:build` to generate it locally, then `pnpm docs:serve` to preview at <http://localhost:3000>.

The hand-written markdown files are also fully readable on their own — open them in your editor or browse them on GitHub if you don't want to run a build.

## Quick build commands

```bash
pnpm docs:build     # Generate the site to .docs-site/
pnpm docs:dev       # Watch mode — rebuild on file changes
pnpm docs:serve     # Local preview at http://localhost:3000
pnpm docs:clean     # Delete .docs-site/
```

For a full reference of contributor commands beyond docs, see the [Build System](contributing/build-system.md) page.
